算法笔记-递归

  1. 1. 引言
  2. 2. 定义
  3. 3. 在计算机科学中的递归

引言

话说对于解释递归，有一句有趣但又十分精准的话：

“要想理解递归，请先理解递归。”

我小学还经常听到个故事：

“从前有座山，山里有座庙，庙里有个小和尚。那个小和尚说：‘从前有座山，山里……’……”（原来我小时候就接触过递归了！）

没错，递归的本质就是套娃（笑 我们能看到一些递归缩写的命名：Linux(Linux is not UNIX), GNU(GNU’s not UNIX), PHP(PHP:Hypertext Preprocessor) 等等，它们就是自己解释自己。

定义

转回正题，下面给出维基百科中关于递归的定义：

“在数学和计算机科学中，递归指由一种（或多种）简单的基本情况定义的一类对象或方法，并规定其他所有情况都能被还原为其基本情况。”

举个栗子，讲递归的话最爱拿的最典型的例子就是“斐波那契数列”了。 我们先看下斐波那契数列的定义：

可以从公式看出，在公式的定义中也包含了自身。

比如当 n = 2 的时候，F(2) = F(1) + F(0) = 1 + 1 = 2。n = 3 的时候 F(3) = F(2) + F(1) = F(1) + F(0) + F(1) = 1 + 1 + 1 = 3…… 所以我们讲斐波那切数列的定义是一个递归定义。 那么，没有n = 0和 n = 1的情况，它是不是也是递归定义呢？ 是的，他仍是一种递归定义，但是这样的话会遇到一种问题，那便是递归的过程永无终结，它会永无止境地调用下去。 所以递归在使用中一定要有终止条件，否则就会一直循环下去不会得到结果（在计算机中最终会使得栈溢出）。

在计算机科学中的递归

在计算机科学中，直接或间接地调用自身的算法称为递归算法。 函数自身给出定义的函数称为递归函数。 使用递归函数所带来的好处便是代码的简洁，在部分算法中使用理解和编写都简单。但是有可能带来效率、性能、空间上的缺失。 在使用递归函数的时候，会经历下面的过程 这个过程中，每次递推都会向系统的栈内存申请空间来存放新的局部变量（压栈），而之前的局部变量因为仍有可能要被使用而不会被销毁。于是就有可能造成栈内存空间不足而造成内存分配失败的情况（StackOverFlow 栈溢出，就是之前提到的）。于是在设计递归函数的时候要注意递归的深度，以防此情况。
